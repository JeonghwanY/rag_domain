# domain.py vs dev_domain.py 비교

## Few-shot 사용 여부

| 파일 | Few-shot | 근거 |
|---|---|---|
| `domain.py` | **없음 (Zero-shot)** | 도메인 정의·규칙만 존재, 정답 예시 없음 |
| `dev_domain.py` | **없음 (Zero-shot)** | 도메인 정의·의도 설명만 존재, 정답 예시 없음 |

두 파일 모두 Zero-shot 기반입니다.

---

## 핵심 차이 요약

| 항목 | domain.py | dev_domain.py |
|---|---|---|
| **접근 방식** | 규칙 기반 (Rule-based) | 의도 기반 (Intent-based) |
| **도메인 설명 구조** | 설명 + 핵심 키워드 | [핵심 의도] + [분류 판단 기준] + [오분류 주의 경계] |
| **페르소나** | 없음 | "10년 베테랑 AIA 상담 팀장" |
| **혼동 방지** | 분류 시 주의사항 (규칙 목록) | 각 도메인의 [오분류 주의 경계] |
| **맥락 독해 원칙** | 없음 | 있음 (4개 원칙) |
| **분류 절차** | 의도 파악 → 주의사항 확인 → 후보 추리 | 의도 파악 → [핵심 의도]/[판단 기준] 확인 → [오분류 경계] 확인 |
| **BATCH_SIZE** | 20 | 20 |

## 도메인 설명 방식 비교

### domain.py
```
### 1. 보험금 보장
보험금 지급 가능 여부 판단, 지급 청구에 필요한 구비서류, ...
핵심 키워드: 보험금 청구, 구비서류, 수술 보장, ...
```
→ **무엇이 해당되는지** 나열하는 방식

### dev_domain.py
```
### 1. 보험금 보장
[핵심 의도]
고객이 특정 보험사고가 발생했을 때 보험 계약이 어떤 조건에서 지급하는지 확인하려는 의도...

[분류 판단 기준]
"이런 상황에서 보장이 되는가"에 집중되어 있다면 이 도메인이다.

[오분류 주의 경계]
- 청구 절차·서류를 묻는다면 → 제지급
- 납입 보험료 금액을 묻는다면 → 보험료납입
```
→ **왜 이 도메인인지, 어떻게 판단하는지** 설명하는 방식

---

## 혼동 방지 방식 비교

### domain.py — 중앙 집중식 주의사항
`DOMAIN_DESCRIPTIONS` 하단에 **분류 시 주의사항** 섹션을 별도로 두고, 자주 혼동되는 케이스를 규칙 목록으로 정리합니다.

```
- 계약정보 vs 보험료납입: 납입주기 변경 → 계약정보 / RTB 납입 방법 → 보험료납입
- 채널 표기 코드 vs 해당 업무: AIA+에서 해약접수 → 채널 표기 코드 / AIA+에서 청약철회 → 제지급
...
```

### dev_domain.py — 도메인 분산식 오분류 경계
각 도메인 정의 안에 **[오분류 주의 경계]** 항목을 두고, 해당 도메인과 혼동될 수 있는 케이스를 각 도메인 내부에서 명시합니다.

```
### 2. 제지급
[오분류 주의 경계]
- 처음으로 보험금을 청구하는 절차를 묻는다면 → 보험금 보장 도메인
- 보장이 되는지 여부 자체를 묻는다면 → 보험금 보장 도메인
```

## 맥락 독해 원칙 (dev_domain.py 전용)

```
- 키워드가 아닌 고객의 궁극적인 목적을 기준으로 분류
- 여러 도메인 키워드 혼재 시 최종 행동 의도 기준으로 판단
- 감정적 표현에 휘둘리지 않고 실제 요청 파악
- 질문이 짧거나 맥락 부족 시 가장 일반적인 상황 전제로 판단
```

## 분류 절차 비교

### domain.py
```
1. 질문의 핵심 의도 파악 (방법/조건/가능여부/정의/절차)
2. 분류 시 주의사항에서 해당 케이스 확인
3. 도메인 정의 보고 후보 1~3개 추린 뒤 최종 선택
```

### dev_domain.py
```
1. 질문의 핵심 의도 파악 (방법/조건/가능여부/정의/절차)
2. [핵심 의도] + [분류 판단 기준] 확인
3. [오분류 주의 경계] 확인하여 후보 1~3개 추린 뒤 최종 선택
```

---

## 언제 어떤 파일을 쓰는가

| 상황 | 권장 파일 |
|---|---|
| 특정 케이스 오분류 규칙 추가·수정 | `domain.py` (주의사항 섹션에 추가) |
| 모델의 맥락 이해력에 기대는 방식 실험 | `dev_domain.py` |
| 운영 배포 | `domain.py` (현재 main.py에 연결됨) |
